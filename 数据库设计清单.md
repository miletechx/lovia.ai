# Lovia.ai 数据库设计清单

根据产品 SPEC，Lovia.ai 第一版数据库需要覆盖用户系统、角色系统、聊天系统、长期记忆、使用额度、会员订阅、图片生成、内容审核和隐私合规。

---

## 1. users 用户表

用于存储注册用户基础信息。

| 字段 | 类型 | 说明 |
|---|---|---|
| id | UUID | 用户唯一 ID |
| email | VARCHAR(255) | 邮箱，唯一，不能为空 |
| username | VARCHAR(50) | 用户名，唯一 |
| name | VARCHAR(100) | 显示名称 |
| avatar_url | TEXT | 用户头像 |
| password_hash | TEXT | 密码哈希值，不保存明文密码 |
| email_verified_at | TIMESTAMP | 邮箱验证时间 |
| age_confirmed | BOOLEAN | 是否确认 18+ |
| subscription_status | VARCHAR(30) | free / premium / canceled / past_due |
| account_status | VARCHAR(30) | active / disabled / deleted |
| last_login_at | TIMESTAMP | 最近登录时间 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

| 用户类型 | 聊天额度 | 图片额度 |
|---|---|---|
| 未登录用户 | 5 条试聊 | 不可用 |
| 登录免费用户 | 每日 30 条 | 不可用或限制使用 |
| 会员用户 | 无限聊天 | 每日 20 张 |

# 密码存储规则

数据库只保存密码哈希：

```text
password_hash
```

禁止保存明文密码。

推荐后端注册流程：

```text
用户输入密码
→ 使用 bcryptjs 或 argon2 生成哈希
→ 保存 password_hash
→ 登录时比对哈希
```

第一版建议使用：

```text
bcryptjs
```
